# F-01 ニュースフィード 要件定義

## 概要

ユーザーが指定した関心カテゴリに沿ってニュースを収集し、AIが不要記事を学習・除外しながらパーソナライズされたフィードを定時配信する機能。

---

## ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | 毎朝、自分の興味あるカテゴリのニュースが届いてほしい |
| US-02 | 「AIの最新動向（ChatGPT, Gemini, Claude Codeなど）」のように具体的なテーマを指定したい |
| US-03 | 大量に届かず、人気のある・重要なニュースに絞ってほしい |
| US-04 | 「このニュースは興味ない」と伝えると、次回から類似記事が除外されてほしい |
| US-05 | 自分の除外条件（趣味嗜好の設定）を確認・編集できるようにしたい |

---

## 機能要件

### 1. ニュース収集

- カテゴリ（または検索クエリ）を設定できる
- 複数カテゴリを登録できる（例: AI動向、テック業界、日本経済）
- 収集対象は「人気のある記事」に絞る（ソーシャルスコア or 掲載量の多さなどで判断）
- 重複記事は1件にまとめる

### 2. ニュース配信

- 定時（例: 毎朝7時）に収集・配信する
- 配信先は当初メール or Slack の1つを選ぶ（両方は後回し）
- 1回の配信あたりのニュース件数は上限を設ける（例: カテゴリあたり5件）

### 3. フィードバック・除外管理

- 「この記事は不要」とフィードバックできる仕組みを持つ（初期はメール返信 or CLIコマンドで可）
- フィードバックを受けて除外条件を保存する
- 除外条件はキーワード・トピックレベルで保存する（例: 「中国規制ニュース」は除外）
- 除外条件の一覧を表示できる
- 除外条件を手動で削除・編集できる

---

## 非機能要件

- 個人利用のため、高可用性は求めない（たまり落ちても許容）
- APIのコストを意識した設計にする（無駄なAPI呼び出しをしない）
- 設定ファイルはバージョン管理できる形式にする

---

## 未決定事項（技術選定で決める）

| 項目 | 候補 | 備考 |
|------|------|------|
| ニュースの取得方法 | RSS / News API / Webスクレイピング | コストと精度を比較 |
| LLM | Claude API / Gemini / OpenAI | 記事の要約・除外判断に使用 |
| 除外条件の保存形式 | JSONファイル / DB / ベクトルDB | 初期はシンプルなファイルでよい |
| 配信手段 | メール / Slack / その他 | まず1つに絞る |
| スケジューラー | cron / Lambda / n8n | ホスティング方針と連動 |

---

## 受け入れ基準（Acceptance Criteria）

- [ ] カテゴリを設定するとそのカテゴリのニュースが収集される
- [ ] 毎朝定時に収集・配信が実行される
- [ ] 1回の配信で件数上限が守られる
- [ ] 「不要」フィードバックを送ると除外条件が保存される
- [ ] 次回配信で除外条件に合致する記事が含まれない
- [ ] 除外条件の一覧を確認できる
